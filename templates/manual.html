How to use the App

Twitter

With Twitter, the App extracts the most recent Tweets that are related to a keyword or a phrase, and then conducts sentiment analysis of the extracted contents.

Required input fields are:

  • Twitter keyword: a keyword or phrase (case insensitive) by which the App searches on Twitter. E.g. movies, loyalty program, etc.
  • Max number of Tweets: for faster computing, smaller numbers are recommended, e.g. 200.
  • Safe mode: when this box is checked, sentiment analysis is conducted locally, i.e. no third party NLP APIs are used. However, analysis performance may be sacrificed to some extent.

Facebook

With Facebook, the user needs to specify an ID of a Facebook post, and then the App extracts the comments to the post. The user may also use keywords/phrases to filter the extracted comments. For example, filtering by "seafood" means that the comments that contain "seafood" are kept for sentiment analysis whereas the rest are discarded.

Required input fields are:

  • Facebook Post ID: the ID of a Facebook post.
  • Max number of posts: for faster computing, smaller numbers are recommended, e.g. 200.
  • Filter by: keywords or phrases for filtering the extracted content. Multiple keywords or phrases may be speficied, which must be separated by comma. E.g. supermarket, Big W, Christmas sales.
  • Safe mode: same as that of Twitter.

OzBargain

With OzBargain, the user needs to specify a keyword and then the App searchs offers/discussions that are relevant to the keyword. Comments on the offers/discussions are extracted and analysed.

Required input fields are:

  • Search keyword: a keyword or phrase by which the App searches on OzBargain, e.g. rewards program.
  • Max number of offers: for faster computing, smaller numbers are recommended, e.g. 10.
  • Max number of comments per offer: max number of comments extracted for each offer/discussion. Avoid using excessively large numbers to reduce computation cost.
  • Safe mode: same as above.
  • Exclude inactive offers: if checked, inactive offers (e.g. those expired or out of stock) will not be included in the analysis.
